# 文章素材管理功能细化

## 1. 项目目标

基于《设计》中的需求，建设一个面向微信公众号运营场景的图片素材采集、筛选、组稿、存草稿、发布一体化系统，核心目标如下：

1. 运营人员在前端输入关键词后，可从多个图片素材网站抓取图片。
2. 抓取后的图片先统一保存到本地素材库，便于重复检索、二次组稿和历史沉淀。
3. 支持在素材库中手动上传本地图片，作为补充素材参与后续组稿。
4. 用户可从素材库中勾选多张图片，自动生成符合微信公众号发布要求的“纯图片欣赏”图文文档格式。
5. 文章生成后先进入系统草稿箱，支持预览、修改、再次编辑。
6. 在草稿箱中一键发布到微信公众号。
7. 图片素材网站支持后续动态扩展，并支持按指定站点或全部站点执行抓取。

---

## 2. 目标用户与使用场景

### 2.1 目标用户

- 微信公众号运营人员
- 内容编辑人员
- 素材管理员
- 系统管理员

### 2.2 典型使用场景

1. 运营输入“春日花海”“治愈系风景”等关键词，批量抓取素材。
2. 系统从多个网站抓取到图片并入库，运营在素材库中筛选可用图片。
3. 运营勾选 10~30 张图片，系统自动生成适合公众号阅读的图文文章。
4. 文章进入草稿箱后，运营做简单调整并发布到微信公众号。
5. 后续再次输入新关键词抓取时，系统继续累计素材，不影响历史数据。

## 3. 系统功能模块

### 3.1 素材来源站点管理

#### 功能目标
对可抓取的图片网站进行统一配置与扩展管理。

#### 功能细项
1. 站点列表展示
   - 站点名称
   - 站点标识
   - 站点域名
   - 抓取状态（启用/停用）
   - 抓取方式说明
   - 最近抓取时间
2. 新增站点
   - 支持录入站点基础信息
   - 支持配置搜索 URL 模板或接口地址
   - 支持配置分页规则
   - 支持配置图片提取规则（如选择器、接口字段映射）
3. 编辑站点
   - 修改站点名称、启用状态、抓取规则
4. 删除/停用站点
   - 停用后不参与抓取任务
5. 测试抓取
   - 输入关键词后验证规则是否可用
   - 返回样例结果，便于排查规则问题

#### 设计建议
- 将站点抓取逻辑做成可插拔结构，后续新增站点时尽量不改动主流程。
- 每个站点保留独立解析配置与错误日志。

---

### 3.2 图片抓取任务

#### 功能目标
根据用户输入关键词，从一个或多个站点抓取图片并入库。

#### 功能细项
1. 抓取任务创建
   - 输入搜索关键词
   - 选择抓取范围：全部站点 / 指定站点
   - 可设置每个站点抓取数量上限
   - 可设置抓取页数上限
2. 任务执行
   - 系统按站点并发或串行抓取
   - 自动提取图片地址、来源链接、标题、标签等信息
3. 图片下载与本地保存
   - 将原图或压缩图下载到本地存储
   - 为图片生成本地访问地址
4. 去重处理
   - 按图片 URL 去重
   - 按图片内容哈希去重
   - 避免同一素材重复入库
5. 任务状态管理
   - 待执行
   - 执行中
   - 已完成
   - 部分成功
   - 失败
6. 任务结果统计
   - 抓取站点数
   - 抓取图片总数
   - 新增入库数量
   - 重复过滤数量
   - 失败数量
7. 失败重试
   - 支持对失败站点单独重试
   - 支持整任务重新执行
8. 抓取日志
   - 记录每个站点抓取耗时
   - 记录错误信息与失败原因

#### 设计建议
- 抓取任务建议异步化，前端通过轮询或 WebSocket 查看进度。
- 对第三方站点增加限流、超时、重试机制，避免因单站异常拖垮整体任务。

---

### 3.3 本地素材库管理

#### 功能目标
沉淀抓取结果，供后续筛选、复用、组稿。

#### 功能细项
1. 素材列表展示
   - 缩略图
   - 标题/关键词
   - 来源站点
   - 来源页面链接
   - 本地存储路径
   - 抓取时间
   - 使用状态
2. 素材搜索与筛选
   - 按关键词搜索
   - 按来源站点筛选
   - 按抓取时间筛选
   - 按使用状态筛选（未使用、已用于文章）
   - 按标签分类展示与筛选
   - 支持按图片名称/素材编号搜索并展示对应爬取记录
3. 素材预览
   - 查看大图
   - 支持点击放大预览与逐张浏览
   - 查看原始尺寸
   - 查看来源信息
   - 查看对应爬取关键词、任务批次与入库时间
4. 标签分类管理
   - 根据每日爬取使用的提示词自动生成标签
   - 支持一张图片绑定多个标签
   - 支持人工补充、修改、合并标签
   - 支持按标签聚合查看素材数量
5. 手动上传图片
   - 支持单张、多张上传
   - 支持 JPG、PNG、WEBP 等常见图片格式
   - 上传时填写标题、关键词、备注等信息
   - 自动生成缩略图并保存到本地素材库
   - 上传来源标记为“手动上传”
6. 素材选择
   - 支持多选、全选、取消选择
   - 支持加入“待组稿清单”
7. 素材标记
   - 收藏
   - 删除
   - 标记不推荐
   - 标记已审核
8. 素材复用
   - 已用素材允许再次用于新文章
9. 素材清理
   - 删除失效图片
   - 删除抓取失败残留文件
   - 清理重复素材

#### 设计建议
- 素材库要区分“物理文件”和“素材记录”，避免直接删库导致文件丢失不可追踪。
- 建议支持缩略图缓存，提高后台浏览性能。

---

### 3.4 文章生成模块

#### 功能目标
将选中的图片素材快速拼装为可发布到微信公众号的文章内容。

#### 功能细项
1. 生成入口
   - 从素材库中勾选多张图片后进入生成页面
2. 发布文档格式生成
   - 根据已选图片生成符合微信公众号发布要求的图文内容结构
   - 输出公众号兼容的正文 HTML
   - 输出可直接用于微信草稿接口的结构化发布数据
   - 支持图片顺序与正文段落一一对应
3. 基础信息设置
   - 文章标题
   - 文章封面（可从已选图片中指定）
   - 作者
   - 摘要
   - 原文链接（可选）
4. 图片排序
   - 支持拖拽排序
   - 支持自动按抓取时间排序
5. 文章正文生成
   - 默认生成“标题 + 多图内容”结构
   - 支持每张图片间插入分隔说明
   - 支持图片下方附带简短描述
   - 支持输出适配微信公众号图文发布的标准正文片段
6. 模板样式
   - 纯图片欣赏模板
   - 极简留白模板
   - 图文混排模板（预留扩展）
7. 内容预览
   - 预览移动端阅读效果
   - 预览公众号图文渲染结构
8. 编辑能力
   - 允许删除单张图片
   - 允许替换图片
   - 允许插入文本段落
9. 保存结果
   - 保存为系统草稿
   - 保存时记录关联素材

#### 设计建议
- 文章正文建议直接生成兼容微信公众号的 HTML 内容。
- 同时建议在后台保留一份结构化发布数据，便于后续直接组装微信公众号草稿/发布接口参数。
- 需要统一处理图片尺寸、居中样式、间距、加载顺序等排版规则。

---

### 3.5 草稿箱管理

#### 功能目标
对生成但未发布的文章进行集中管理。

#### 功能细项
1. 草稿列表
   - 标题
   - 封面
   - 创建时间
   - 最后修改时间
   - 发布状态
2. 草稿预览
   - 支持文章卡片预览
   - 支持进入详情页查看完整公众号文章预览效果
   - 支持预览封面、标题、摘要、正文图片顺序与排版
   - 支持预览公众号样式与移动端阅读效果
3. 草稿编辑
   - 修改标题、摘要、正文
   - 调整图片顺序
   - 新增/删除图片
4. 草稿复制
   - 基于旧草稿生成新文章
5. 草稿删除
   - 支持软删除
6. 草稿发布
   - 调用微信公众号接口发布
7. 发布结果反馈
   - 成功：记录公众号素材 ID / 发布结果
   - 失败：记录错误原因，允许再次发布

#### 草稿状态建议
- 编辑中
- 待发布
- 发布中
- 已发布
- 发布失败

---

### 3.6 微信公众号发布模块

#### 功能目标
与微信公众号平台对接，实现草稿文章发布。

#### 功能细项
1. 公众号配置管理
   - AppID
   - AppSecret
   - Token 或相关接口认证配置
2. 素材上传
   - 将文章中使用的本地图片上传到微信素材接口或按需要处理
3. 草稿同步
   - 调用微信草稿相关接口创建草稿
4. 正式发布
   - 调用发布接口完成推送/群发/发布动作
5. 发布记录管理
   - 发布时间
   - 发布账号
   - 微信返回结果
   - 发布状态
6. 失败处理
   - Token 失效提醒
   - 素材上传失败重试
   - 敏感词/内容审核失败提示

#### 设计建议
- 发布接口应与“文章生成”解耦，避免生成环节强依赖微信接口可用性。
- 需要封装统一的微信 API 客户端与错误处理机制。

## 4. 核心业务流程

### 4.1 素材抓取流程

1. 用户输入关键词。
2. 用户选择抓取范围（全部站点或指定站点）。
3. 系统创建抓取任务。
4. 各站点执行搜索并解析结果。
5. 图片下载到本地存储。
6. 系统进行去重、入库、缩略图处理。
7. 返回抓取结果并展示在素材库中。

### 4.2 手动上传入库流程

1. 用户在素材库选择手动上传。
2. 选择本地图片文件并填写基础信息。
3. 系统校验图片格式、大小、分辨率等规则。
4. 图片保存到本地存储并生成缩略图。
5. 系统写入素材记录，来源类型标记为手动上传。
6. 上传完成后图片可直接参与筛选和组稿。

### 4.3 每日标签分类流程

1. 系统按天归集当日抓取任务与提示词。
2. 根据抓取关键词、提示词和来源站点为图片自动生成标签。
3. 对同义标签进行归并，形成分类目录。
4. 素材库按标签更新分类展示结果。
5. 用户可在后台人工修正标签与分类关系。

### 4.4 文章生成流程

1. 用户在素材库筛选和勾选多张图片。
2. 用户可按标签分类浏览素材，并查看单图对应爬取记录后确认选图。
3. 进入文章生成页面。
4. 设置标题、封面、摘要等信息。
5. 调整图片顺序与模板。
6. 系统生成公众号兼容 HTML 与结构化发布数据。
7. 保存到草稿箱。

### 4.5 发布流程

1. 用户进入草稿箱。
2. 预览并确认由多图生成的公众号发布文档内容。
3. 系统上传必要图片素材到微信侧。
4. 系统按微信要求组装草稿或发布接口数据。
5. 系统创建微信草稿或直接执行发布。
6. 记录发布结果与日志。

## 6. 企业级规划建议

### 6.1 推荐代码目录结构
- [`src/modules/material`](src/modules/material)
  - [`controller`](src/modules/material/controller)

- [`src/modules/tag`](src/modules/tag)
- [`src/common`](src/common)
  - [`config`](src/common/config)
  - [`exception`](src/common/exception)
  - [`middleware`](src/common/middleware)
  - [`utils`](src/common/utils)

### 6.2 字段命名规范
- 数据对象统一采用英文语义化命名。
- 主键统一使用 `id`，外键统一使用 `xxx_id`。
- 状态字段统一使用 `xxx_status` 或 `enabled`。
- 时间字段统一使用 `created_at`、`updated_at`、`published_at`、`crawled_at`。
- 编码类字段统一使用 `xxx_code`，名称类字段统一使用 `xxx_name`。
- 文本说明类字段统一使用 `summary`、`remark`、`caption_text`、`prompt_text` 等明确语义字段名。

### 6.3 模块职责建议
- 素材模块负责图片入库、预览、上传、标签、爬取记录查询。
- 抓取模块负责站点适配、任务调度、抓取日志、失败重试。
- 文章模块负责多图组稿、正文生成、草稿管理、文章预览。
- 发布模块负责微信素材上传、草稿同步、正式发布、发布结果回写。
- 标签模块负责按每日提示词自动打标签、标签归并、分类展示与检索。

## 7. 后台能力建议

### 7.1 调度与任务队列
- 抓取任务建议进入任务队列执行。
- 支持并发站点抓取。
- 支持失败重试和超时控制。

### 7.2 文件存储
- 原始图片本地存储。
- 缩略图单独目录存储。
- 预留后期迁移到对象存储的能力。

### 7.3 去重能力
- URL 去重。
- 文件哈希去重。
- 可选相似图片识别去重。

### 7.4 安全与风控
- 限制单次抓取频率，避免目标网站封禁。
- 增加 User-Agent、代理、重试间隔等策略。
- 对外部图片内容做基础校验，避免异常文件入库。

### 7.5 可维护性
- 每个站点单独适配器。
- 微信接口统一服务封装。
- 任务日志与业务日志分离。

---

## 8. 非功能需求建议

### 8.1 性能
- 单次抓取任务应支持数十到数百张图片处理。
- 素材列表应支持分页、懒加载。

### 8.2 稳定性
- 站点解析失败不应影响其他站点任务。
- 发布失败不影响草稿保存。

### 8.3 可扩展性
- 支持后续新增更多素材网站。
- 支持未来扩展为图文混排、AI 生成标题、自动摘要等能力。

### 8.4 可用性
- 关键节点要有明确状态提示。
- 长任务要可视化进度。
- 错误信息要可定位、可重试。

- 生成文章
- 管理草稿
- 发起发布

## 10. MVP 最小可用版本建议

### 第一阶段必须实现
1. 关键词抓取图片
2. 指定站点/全部站点抓取
3. 图片本地入库
4. 素材库手动上传图片
5. 素材库图片放大预览
6. 素材库按标签分类展示与筛选
7. 按图片名称/素材编号查看对应爬取记录
8. 根据所选图片生成公众号文章 HTML 与发布结构化数据
9. 草稿箱文章预览
10. 保存草稿
11. 调用微信公众号接口发布

### 第二阶段建议增强
1. 站点规则后台配置化
2. 每日提示词自动打标签与标签归并
3. 图片审核与收藏
4. 模板样式丰富化
5. 发布失败自动重试
6. 数据统计看板

### 第三阶段扩展方向
1. AI 自动生成标题和摘要
2. AI 智能排序图片
3. 多公众号账号切换发布
4. 云存储迁移
5. 敏感内容校验

---

## 11. 风险与注意事项

1. 第三方图片网站结构变化频繁，需要做好适配器维护机制。
2. 抓取行为需关注目标网站的访问规则与合规性。
3. 微信公众号接口调用受限于官方平台能力与频率限制。
4. 图片版权问题需在业务上明确使用边界。
5. 图片过多时，文章内容体积可能过大，需要压缩与尺寸控制。

---

## 12. 推荐实施顺序

1. 先完成站点抓取适配 + 本地素材入库。
2. 再完成素材库筛选与文章生成。
3. 再对接微信公众号草稿/发布接口。
4. 最后完善站点管理、统计分析、日志和权限系统。
